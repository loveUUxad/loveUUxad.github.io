dubbo是阿里巴巴下一个开源的服务框架，其本质是一个资源调度和治理中心，在分布式项目中，各个模块之间的接口需要相互调用，如果采用webService的URL式声明服务地址，随着服务的增多，我们的配置也会增多，不利于管理和维护，调用起来也十分麻烦，而dubbo则很好的解决了这一弊端，dubbo传输协议采用的是RPC协议，这种协议底层采用的是socket流传输文件，在速度上也较http协议有优势,所以被现在很多公司广泛使用。

dubbo的工作原理

Provider

暴露服务方称之为“服务提供者”。

Consumer

调用远程服务方称之为“服务消费者”。

Registry

服务注册与发现的中心目录服务称之为“服务注册中心”,这里我们一般试用zookeeper作为他的注册中心

Monitor

统计服务的调用次数和调用时间的日志服务称之为“服务监控中心”，使用的最多的是dubbo-admin这款开源项目

总结来说服务提供方在启动服务的时候向注册中心注册自己的服务，消费方只需要去订阅自己所需要的服务即可，服务的调用都会被给我们监控中心所记录

简单入门

注册中心的单机版安装

这里我们采用的是zookeeper,点击这里下载zookeeper的安装包

  • 上传到服务器上 解压到当前目录

  • 进入到解压目录的conf文件夹，并将zoo_sample.cfg改名为zoo.cfg,因为zookeeper自动识别的是zoo.cfg,然后编辑zoo.cfg

    更改我的数据存放路径(这里可以自己指定)
    dataDir=/usr/local/zookeeper-3.4.12/data
    更改我的日志存放路径
    dataLogDir=/usr/local/zookeeper-3.4.12/logs
    默认端口号(这里我改为2185)
    clientPort=2185
    保存退出

  • 启动
    进入安装目录下bin/下启动zkServer.sh start，启动成功，查看其状态为standlone单例模式

监控中心中心在windows系统下的安装

在网上下载dubbo-admin.war,然后放到本地tomcat的webapps下面，启动tomcat会进行自动编译，之后进入编译后的dubbo-admin/WEB_INF目录下，编辑dubbo.properties，更改注册中心ip端口,并设置用户名密码，这里我设置了用户guest和root，密码和用户名都一样


再次启动tomcat访问locathost:8080/dubbo-admin-2.6/ 并登录就可以看到我们的界面了，点击service就可以看到我们的服务了
